# Demo128-NAND Flash 存储结构可视化

## 1. 物理分区布局图

```

┌─────────────────────────────────────────────────────────────┐
│                                                               │
│ ┌──────────────────────────────────────────────────────────┐ │
│ │ 0x0000000 (0B)                                           │ │
│ │ ┌────────────────────────────────────────────────────┐   │ │

│ │ │ 0x0100000 (1MB)                                         │ │
│ │ ┌────────────────────────────────────────────────────┐   │ │

│ │ │ 地址：0x0100000 - 0x0140000                       │   │ │
│ │ │ 用途：U-Boot 环变存储                             │   │ │
│ │ │ 特点：与 env_r 冗余备份                           │   │ │

│ │ │ 地址：0x0140000 - 0x0180000                       │   │ │
│ │ │ 用途：环变冗余备份                                │   │ │
│ │ │ 恢复：主 env 损坏时使用                           │   │ │

│ │ │ 0x0F80000 (15.75MB)                                     │ │
│ │ ┌────────────────────────────────────────────────────┐   │ │

│ │ │ 用途：FAT 文件系统 (rodata.fatfs)                 │   │ │

│ │ │ 包含：.mo i18n 文件、图片、配置等                 │   │ │
│ │ │ 特点：应用启动时完全加载到内存                    │   │ │
│ │ └────────────────────────────────────────────────────┘   │ │
│ │ │ 0x3880000 (57.75MB)                                     │ │
│ │ ┌────────────────────────────────────────────────────┐   │ │

│ │ │ 地址：0x3880000 - 0x6280000                       │   │ │

│ │ ├─ 用户应用数据                                     │   │ │
│ │ ├─ 日志文件                                         │   │ │
│ │ └─ 缓存文件                                         │   │ │
│ │ │ 特点：                                             │   │ │
│ │ │ • NFTL 层：自动磨损平衡、坏块管理               │   │ │
│ │ │ • 文件系统：FATFS 提供标准接口                   │   │ │
│ │ │ • 掉电保护：NAND Flash 本身属性                  │   │ │
│ │ │ • 恢复能力：NFTL 垃圾回收                        │   │ │

│ │ │ 用途：坏块预留、管理信息、垃圾回收               │   │ │
│ │ │ 分配：NFTL 智能管理                               │   │ │
│ │ └────────────────────────────────────────────────────┘   │ │

│ │ └──────────────────────────────────────────────────────────┘ │
```

```
应用程序层
┌────────────────────────────────────────────────┐
│  xnapp/gui 应用                                │
│                                                │
│  json_get_strvalue("skinpng_path")            │
│  json_set_intvalue("current_page", 2)         │
│  json_set_strvalue("device_connect", "ok")    │
│  ...                                           │
└────────────────┬─────────────────────────────┘
                 │ (函数调用)
                 ↓
JSON 管理模块 (app_config.c)
┌────────────────────────────────────────────────┐
│ g_json_root (cJSON 树)    [内存中]             │
│                                                │
│ {                                              │
│   "skinpng_path": "",                         │
│   "skin_circle_id": 0,                        │
│   "current_page": 0,                          │
│   "device_connect": 0,                        │
│   "pre_screen": 0                             │
│ }                                              │
│                                                │
│ 特点：                                         │
│ • 快速访问（RAM 操作）                        │
│ • 互斥保护（g_json_mutex）                    │
│ • 实时更新（内存立即反映）                    │
└────────────────┬─────────────────────────────┘
                 │ (定时触发每30秒)
                 ↓
定时保存线程 (save_task)
┌────────────────────────────────────────────────┐
│ void save_task(void *parameter) {             │
│     while (1) {                               │
│         rt_thread_mdelay(30000);              │
│         json_save_file(JSON_FILE_PATH);       │
│     }                                         │
│ }                                              │
└────────────────┬─────────────────────────────┘
                 │ (序列化)
                 ↓
序列化为字符串
┌────────────────────────────────────────────────┐
│ char *json_str = cJSON_Print(g_json_root);    │
│                                                │
│ 结果：                                         │
│ "{\"skinpng_path\":\"\",\"skin_circle_id\"... │
│                                                │
│ 大小：通常 < 1KB                             │
└────────────────┬─────────────────────────────┘
                 │ (打开文件)
                 ↓
文件 I/O 操作
┌────────────────────────────────────────────────┐
│ fd = open("/data/config.json",                │
│           O_WRONLY | O_CREAT | O_TRUNC);      │
│ write(fd, json_str, strlen(json_str));        │
│ close(fd);                                    │
│                                                │
│ 特点：                                         │
│ • O_TRUNC：覆盖写入（保证原子性）            │
│ • close() 确保数据落盘                       │
└────────────────┬─────────────────────────────┘
                 │ (文件系统调用)
                 ↓
FATFS 文件系统层
┌────────────────────────────────────────────────┐
│ • 查找 /data 目录                             │
│ • 创建或更新 config.json 目录项              │
│ • 分配簇链存储文件数据                        │
│ • 更新 FAT 表和目录表                        │
│ • 标记簇为已用                                │
│                                                │
│ /data 目录结构：                              │
│ ├─ config.json (新增/更新)                   │
│ ├─ [其他文件]                                │
│ └─ [FAT 表元数据]                            │
└────────────────┬─────────────────────────────┘
                 │ (NAND 驱动)
                 ↓
NFTL 翻译层 (NAND Flash Translation Layer)
┌────────────────────────────────────────────────┐
│ 输入：逻辑块地址 (LBA)                        │
│ 处理：                                         │
│  1. 地址映射：LBA → PBA (物理块地址)         │
│  2. 磨损平衡：选择写入次数最少的块            │
│  3. 坏块检测：跳过已知坏块                    │
│  4. 垃圾回收：后台清理无效数据                │
│                                                │
│ 映射表：                                       │
│ ┌─────────────────┐                           │
│ │ LBA  │ PBA │ 擦次 │                         │
│ ├──────┼─────┼─────┤                         │
│ │ 100  │ 234 │ 45  │                        │
│ │ 101  │ 567 │ 42  │  → 选择此块 (擦次少)   │
│ │ 102  │ 812 │ 48  │                        │
│ └─────────────────┘                           │
│                                                │
│ 输出：转换后的物理地址                        │
└────────────────┬─────────────────────────────┘
                 │ (NAND 物理写入)
                 ↓
NAND Flash 物理存储 (data 分区)
┌────────────────────────────────────────────────┐
│ 地址：0x3880000 - 0x6280000 (50MB)            │
│                                                │
│ 物理写入步骤：                                 │
│ 1. 编程：在物理块中写入 config.json 数据     │
│ 2. 验证：读回数据验证是否正确                │
│ 3. ECC：计算 ECC 校验码存储到 OOB 区        │
│ 4. 完成：标记块为有效数据                    │
│                                                │
│ 掉电保护：                                     │
│ ✓ NAND Flash 无电源依赖                      │
│ ✓ 数据立即持久化                             │
│ ✓ 断电重启后数据完整恢复                     │
│                                                │
│ 特点：                                         │
│ • 每块 128KB，每页 2KB                       │
│ • 支持 100万+ 擦写周期                       │
│ • OOB (64字节) 存储 ECC 和坏块标记          │
└────────────────────────────────────────────────┘

恢复流程（系统重启）：
                 ↓
NAND 驱动读取
        ↓
NFTL 地址转换
        ↓
FATFS 目录遍历
        ↓
加载 /data/config.json
        ↓
cJSON_Parse() 解析
        ↓
g_json_root 重建
        ↓
应用可继续访问
```

## 3. 系统启动序列时序图

```
时间轴 ────────────────────────────────────────────────────→

[系统开机]
     │
     ├─ DDR 初始化
     │  └─ 加载 ddr_init.bin (来自 SPL)
     │
T0:  ├─ 读取 SPL 分区 (1MB)
     │  └─ 执行 Bootloader
     │
T1:  ├─ 读取 ENV/ENV_R 分区 (512KB)
     │  └─ 加载环境变量 (4KB 有效数据)
     │
T2:  ├─ 读取 OS 分区 (15MB)
     │  └─ 内核启动
     │
T3:  ├─ 读取 RODATA 分区 (35MB)
     │  └─ 挂载 /rodata (FAT 只读)
     │  └─ 加载 .mo 文件到 RAM
     │
T4:  ├─ 读取 DATA 分区 (50MB)
     │  └─ 挂载 /data (FAT 读写)
     │
T5:  ├─ 初始化 JSON 管理模块
     │  ├─ 创建互斥锁 g_json_mutex
     │  ├─ 加载 /data/config.json
     │  └─ 启动定时保存线程
     │
T6:  └─ 系统完全启动
          应用程序可运行
          
[运行时操作]
T6+:  应用修改 config.json (内存操作 - 快速)
      ├─ json_set_intvalue("current_page", 2)
      ├─ json_set_strvalue("device_connect", "ok")
      └─ 立即反映到 g_json_root
      
T6+30s: [定时保存触发]
        ├─ 后台线程序列化为字符串
        ├─ 写入 /data/config.json (文件系统)
        ├─ FATFS 分配簇链
        ├─ NFTL 映射物理地址
        ├─ NAND 编程并验证
        └─ 数据保存完成
        
T6+60s: [再次定时保存] (同上)
        
[掉电场景]
T_crash: 系统突然断电
         ├─ 内存中 g_json_root 丢失（可接受）
         ├─ NAND Flash 中 config.json 完整保存
         └─ 无需担心数据丢失
         
[重启]
     ├─ 重复 T0-T5 流程
     └─ json_load_file() 从 NAND 重新读取 config.json
```

## 4. NFTL 磨损平衡演示图

```
场景：频繁写入 config.json

初始状态（擦写次数）：
┌─────────────────────────────────────────┐
│ 块号 │ 0  │ 1  │ 2  │ 3  │ 4  │ 5  │ ... │
├─────┼────┼────┼────┼────┼────┼────┤────│
│擦数 │ 45 │ 42 │ 48 │ 40 │ 50 │ 44 │ ...│
└─────────────────────────────────────────┘

[config.json 第1次写入]
NFTL 选择块 3（擦次 40 - 最少）
        ↓
┌─────────────────────────────────────────┐
│ 块号 │ 0  │ 1  │ 2  │ 3* │ 4  │ 5  │ ...│
├─────┼────┼────┼────┼────┼────┼────┤────│
│擦数 │ 45 │ 42 │ 48 │ 41 │ 50 │ 44 │ ...│  (* 新增擦写)
└─────────────────────────────────────────┘

[config.json 第2次写入]
NFTL 选择块 1（擦次 42 - 现在最少）
        ↓
┌─────────────────────────────────────────┐
│ 块号 │ 0  │ 1* │ 2  │ 3  │ 4  │ 5  │ ...│
├─────┼────┼────┼────┼────┼────┼────┤────│
│擦数 │ 45 │ 43 │ 48 │ 41 │ 50 │ 44 │ ...│
└─────────────────────────────────────────┘

[config.json 第3次写入]
NFTL 选择块 3（擦次 41 - 再次最少）
        ↓
┌─────────────────────────────────────────┐
│ 块号 │ 0  │ 1  │ 2  │ 3* │ 4  │ 5  │ ...│
├─────┼────┼────┼────┼────┼────┼────┤────│
│擦数 │ 45 │ 43 │ 48 │ 42 │ 50 │ 44 │ ...│
└─────────────────────────────────────────┘

[继续写入...]

最终结果：
所有块的擦写次数均衡分布（都在 40-50 之间）
        ↓
✓ 没有单个块过度磨损
✓ Flash 整体寿命最大化
✓ 即使高频写入也能保证长期使用

对比（无 NFTL）：
块 3 被频繁选中 → 擦写次数 > 100万 → 提前损坏
块 0 几乎不用   → 擦写次数 < 10万  → 浪费
结果：Flash 寿命缩短 50% 以上
```

## 5. 坏块管理流程

```
NAND Flash 出现坏块

     初始化时
          │
     检测坏块标记
          │
    构建坏块表 (BBT)
          │
     记录到内存
          ↓
┌────────────────────────────────────────┐
│ 坏块表 (BBT)                           │
│ ┌────────────────────────────────────┐ │
│ │ 块号 │ 状态    │ 处理               │ │
│ ├──────┼─────────┼──────────────────┤ │
│ │ 50   │ 好块    │ 正常使用          │ │
│ │ 51   │ 好块    │ 正常使用          │ │
│ │ 52   │ 坏块    │ 跳过（不使用）    │ │ ← 出厂坏块
│ │ 53   │ 好块    │ 正常使用          │ │
│ │ ...  │ ...     │ ...               │ │
│ │ 200  │ 坏块    │ 跳过（不使用）    │ │ ← 运行时坏块
│ └────────────────────────────────────┘ │
└────────────────────────────────────────┘

写入流程（地址转换）：

用户请求：写入逻辑块 100
        │
    查询坏块表
        │
    块 100 是好块? → 是
        │
  物理写入块 100
        │
  完成（无需转向）

用户请求：写入逻辑块 200
        │
    查询坏块表
        │
    块 200 是好块? → 否（坏块）
        │
  转向到替换块（如 250）
        │
  物理写入块 250
        │
  记录映射：200 → 250
        │
  完成（透明处理）

读取流程：

用户请求：读取逻辑块 200
        │
    查询坏块表
        │
    块 200 是好块? → 否
        │
  查询映射表：200 → 250
        │
  物理读取块 250
        │
  返回数据（用户无感知）

垃圾回收：

NFTL 后台运行
        │
  检测过度磨损的块
        │
  检测将要失效的块
        │
  标记为坏块
        │
  转向到新块
        │
  更新映射表和坏块表
        │
  完成（对用户透明）
```

## 6. 数据恢复流程

```
场景：config.json 分区数据损坏

[损坏检测]
         │
    应用读取 config.json
         │
    FATFS 错误（簇链中断）
         │
    NFTL 错误检测（ECC 失败）
         │
    标记块为坏块
         │
         ↓

[恢复启动]
         │
    NFTL 垃圾回收线程
         │
    扫描所有有效数据块
         │
    查找 config.json 的有效副本
         │
  （因为 NFTL 内部维护多个副本）
         │
    恢复数据到新块
         │
    更新坏块表
         │
    更新映射表
         │
         ↓

[恢复完成]
         │
    下次读取时
         │
    自动从恢复块读取
         │
    应用继续运行
         │
    用户无感知

特殊情况：如果 NFTL 无法恢复

    应用启动时检测
         │
    json_load_file() 返回 NULL
         │
    json_manager_init() 重建默认配置
         │
    cJSON_CreateObject()
         │
    cJSON_AddStringToObject(root, "name", "default")
         │
    json_save_file() 写入新配置
         │
    系统继续运行（配置重置）
```

## 7. config.json 内容示例与存储分析

```
实际 config.json 文件：
┌────────────────────────────────────────────────────────┐
│ {                                                       │
│     "skinpng_path": "",                                │
│     "skin_circle_id": 0,                               │
│     "current_page": 0,                                 │
│     "device_connect": 0,                               │
│     "pre_screen": 0                                    │
│ }                                                       │
│                                                         │
│ 序列化后大小：约 120 字节                              │
└────────────────────────────────────────────────────────┘

存储分配：

NAND Flash 中
    │
    └─ /data 分区 (50MB)
       │
       ├─ FATFS 文件系统元数据
       │  ├─ FAT 表 (多个副本) - 约 64KB
       │  ├─ 根目录表 - 约 1KB
       │  └─ 保留扇区 - 约 32KB
       │
       ├─ config.json (120 字节)
       │  ├─ 实际数据：120 字节
       │  ├─ 分配簇数：1 (4096 字节/簇)
       │  ├─ 物理块：1 个 128KB 块的一部分
       │  ├─ ECC 冗余：128 字节 (在 OOB 中)
       │  └─ NFTL 内部副本：至少 2 份
       │
       ├─ [其他应用数据]
       │
       └─ [NFTL 管理信息]
          ├─ 坏块表 (BBT)
          ├─ 映射表
          └─ 擦写计数器

更新周期分析：

每次更新
    │
T = 0: 应用修改内存中的 JSON
       消耗：< 1ms (RAM 操作)
       存储：0 (未保存)
       风险：掉电丢失
    │
T = 30s: 定时保存触发
       步骤 1: 序列化 (< 1ms)
       步骤 2: 打开文件
       步骤 3: 写入数据到 FATFS
       步骤 4: FATFS 分配簇
       步骤 5: 通知 NFTL
       步骤 6: NFTL 地址转换 (< 5ms)
       步骤 7: 物理写入 (< 50ms)
       步骤 8: ECC 计算并存储
       步骤 9: 关闭文件
    │
总耗时：约 60-100ms
运行时影响：可接受（后台进行）
    │
存储结果：
    ├─ 主副本写入完成
    ├─ NFTL 隐含备份
    ├─ FAT 表更新
    └─ 数据永久化

在 30 秒内的多次修改会合并保存
以降低 Flash 写入频率
```
