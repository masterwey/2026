# 存储空间分配详细参数表

## 表格1：完整分区参数对照表

| 序号 | 分区名 | 大小 | 十六进制地址 | 物理地址范围 | 属性 | 文件格式 | 冗余备份 | 用途说明 |
|------|--------|------|------------|-------------|------|---------|---------|---------|
| 1 | **spl** | 1MB | 0x0 ~ 0x100000 | 0 ~ 1048576 | MTD(R) | .aic | ✗ | 一级引导(SPL/Bootloader)，系统冷启动 |
| 2 | **env** | 256K | 0x100000 ~ 0x140000 | 1048576 ~ 1310720 | MTD(R) | env.bin | ✓ env_r | U-Boot 环变，冗余存储 |
| 3 | **env_r** | 256K | 0x140000 ~ 0x180000 | 1310720 ~ 1572864 | MTD(R) | env.bin | ✓ env | 环变备份，主 env 损坏时恢复 |
| 4 | **os** | 15MB | 0x180000 ~ 0xF80000 | 1572864 ~ 16252928 | MTD(R) | d21x_os.itb | ✓ os_r | 操作系统镜像(Linux/RT-Thread) |
| 5 | **os_r** | 15MB | 0xF80000 ~ 0x1D80000 | 16252928 ~ 30932992 | MTD(R) | d21x_os.itb | ✓ os | OS 冗余备份，fallback 启动 |
| 6 | **rodata** | 35MB | 0x1D80000 ~ 0x3880000 | 30932992 ~ 59768832 | MTD(RO) | rodata.fatfs | ✗ | FAT 只读资源(图片/.mo 文件) |
| 7 | **data** | 50MB | 0x3880000 ~ 0x6280000 | 59768832 ~ 104857600 | Block(RW) | data.fatfs | ◆ NFTL | FAT+NFTL 读写数据(**config.json**) |
| - | **预留** | ~11.5MB | 0x6280000 ~ 0x8000000 | 104857600 ~ 134217728 | 系统 | - | - | 坏块预留、管理信息、垃圾回收 |

**属性说明：**
- `MTD(R)` = MTD 驱动，只读（使用原始驱动读取）
- `MTD(RO)` = MTD 驱动，只读FAT文件系统
- `Block(RW)` = 块设备，读写（支持文件系统）
- `✓` = 具有此冗余备份
- `✗` = 无冗余备份
- `◆ NFTL` = NFTL 层隐含备份（通过垃圾回收）

## 表格3：config.json 存储生命周期

| 时间 | 操作 | 位置 | 状态 | 风险 | 说明 |
|------|------|------|------|------|------|
| **T=0s** | 应用启动 | RAM | 从 /data/config.json 加载 | 低 | 系统初始化 |
| **T=0.1s** | json_manager_init() | RAM | 创建 cJSON 树 | 低 | 初始化互斥锁 |
| **T=0.5s** | 应用修改 JSON | RAM | cJSON 树内存更新 | **高** ⚠ | 仅在 RAM，掉电丢失 |
| **T=1s** | 继续运行... | RAM | 多次修改积累 | **高** ⚠ | 最后修改距离保存最远 |
| **T=30s** | 定时器触发 | RAM+NAND | 序列化 + 写入 | 低 | 开始保存流程 |
| **T=30.05s** | json_save_file() | NAND | 文件 I/O 完成 | 低 | FATFS 层分配簇 |
| **T=30.1s** | NFTL 地址转换 | NAND | 物理地址映射 | 低 | 磨损平衡选块 |
| **T=30.15s** | MTD 编程 | NAND Flash | ECC 计算 | 低 | 物理写入 |
| **T=30.2s** | close(fd) 返回 | NAND Flash | ✓ 数据落盘 | **无** ✓ | 持久化完成 |
| **T=30.2s ~ 60s** | 继续运行... | RAM | 新一轮修改 | **高** ⚠ | 等待下次保存 |
| **T=60s** | 再次定时保存 | NAND Flash | (重复 T=30s 流程) | 低 | 周期性保存 |

**关键时间点：**
- **掉电高风险区** = T=0.5s ~ T=30s (共 29.5s)
- **掉电低风险区** = T=30.2s ~ T=60s (共前 30s)
- **最长丢失数据** = 最多 30 秒内的修改

## 表格7：故障场景与恢复

| 故障类型 | 发生位置 | 检测 | 恢复策略 | 数据丢失 | 恢复时间 |
|---------|---------|------|---------|---------|---------|
| **SPL 损坏** | spl 分区 | 启动失败 | 无法恢复 → 砖块 | 全部 | ∞ |
| **ENV 损坏** | env 分区 | 加载失败 | 自动使用 env_r | 无 | < 1s |
| **OS 损坏** | os 分区 | 启动失败 | 自动启动 os_r | 无 | < 1s |
| **config.json 损坏(已保存)** | data 分区 | 读失败/ECC 错 | NFTL 恢复 + 默认值 | < 30s | 1-2s |
| **config.json 损坏(未保存)** | data 分区 | 无法检测 | 掉电后重建默认 | 上次保存后的修改 | 重启 |
| **坏块产生** | data 分区 | NFTL 后台检测 | 自动转向新块 | 无 | 透明 |
| **NAND 多块故障** | data 分区 | ECC 连续失败 | NFTL 垃圾回收(可能失败) | 可能全部 | - |

## 表格8：掉电保护对比

### 不同层次的掉电安全性

| 层级 | 是否安全 | 说明 | 恢复能力 |
|------|---------|------|---------|
| **内存(RAM)** | ✗ 完全丢失 | 所有未保存修改丢失 | 需重新输入 |
| **FATFS 目录** | ◆ 部分安全 | FAT 表可能不一致 | 文件系统检查修复 |
| **NFTL 层** | ✓ 高度安全 | 自动数据恢复 | 垃圾回收机制 |
| **NAND Flash** | ✓ 完全安全 | 物理存储不受电源影响 | 无需恢复 |

### config.json 的掉电情景

```
场景 1：在 T=5s 时掉电（未保存）
  结果：内存中的修改全部丢失
  恢复：系统重启后加载上次保存的 config.json
  数据丢失：最多 30 秒内的修改

场景 2：在 T=35s 时掉电（已保存）
  结果：文件系统层修改可能中断
  恢复：NFTL 自动检测并恢复
  数据丢失：0 秒（已完全保存）

场景 3：多块 NAND 故障
  结果：config.json 可能彻底丢失
  恢复：系统使用默认配置重启
  数据丢失：整个 config.json
```

## 总结对比表

### 分区分类

| 分类 | 分区 | 特点 | 可靠性 |
|------|------|------|--------|
| **系统关键** | SPL/OS | 必需、不可恢复 | 高(冗余备份) |
| **系统配置** | ENV | 必需、可恢复 | 高(冗余) |
| **应用数据** | data | 可选、频繁修改 | 中(NFTL 保护) |
| **静态资源** | rodata | 可选、只读 | 中(可重新制作) |

### 保护方式

| 方式 | 分区 | 机制 | 有效性 |
|------|------|------|--------|
| **硬件级** | 全部 | NAND Flash 本身属性 | ✓ 最强 |
| **冗余备份** | env/os | 双分区 | ✓ 强 |
| **NFTL** | data | 磨损平衡+垃圾回收 | ✓ 中等 |
| **软件级** | config.json | 定时保存 | ◆ 可以改进 |
