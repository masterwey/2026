## 🎯 核心发现总结

### 存储架构设计

```
128MB SPI-NAND Flash 分配：
├─ SPL (1MB)           ← 启动代码
├─ ENV (256K) + 备份   ← 环境变量（冗余）
├─ OS (15MB) + 备份    ← 操作系统（冗余）
├─ RODATA (35MB)       ← 资源和 .mo 文件
├─ DATA (50MB)         ← ★ config.json ★
└─ 预留 (11.5MB)       ← 坏块和管理
```

### config.json 掉电安全保证

**三层保护机制：**

```
Layer 1: NAND Flash 硬件特性
         └─ 掉电后数据不丢失 (最强)

Layer 2: NFTL 翻译层
         ├─ 磨损平衡：自动选择最优块
         ├─ 坏块管理：透明处理
         └─ 垃圾回收：后台恢复 (强)

Layer 3: 定时保存
         ├─ 周期：30秒
         ├─ 方式：后台线程
         └─ 风险：最多丢失30s修改 (中等)
```

### .mo 文件处理

```
xnapp.mo (国际化文件)
    ↓
打包到 rodata 分区 (35MB FATFS)
    ↓
系统启动时加载到 RAM
    ↓
应用运行时通过 gettext 库访问
```

### NFTL 的关键作用

```
问题：NAND Flash 写入不均匀导致早期损坏

无 NFTL:
  最常用块：>100万次擦写 → 损坏
  最少用块：<10万次擦写  → 浪费
  结果：实际寿命 ~50% 缩短

有 NFTL:
  所有块：~均衡分布 (~10万次/块)
  结果：实际寿命 ~10 倍提升 ✓
```
